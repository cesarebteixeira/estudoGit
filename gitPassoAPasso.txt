Git

1 - criar uma pasta. A qual será o nosso repositório;
2 - depois de instalar o git e suas configurações...
3 - ir para a pasta criada para ser o repositório;
4 - git init >.agora temos um respositório controlado pelo git
5 - copiamos dois arquivos_arq1.txt e arq2.txt) textos para a pasta do repositório
6- git status > git informa que temos 2 arquivos untracked . Ou seja os arquivos não estão rastreaveis
7 - git add arq1.txt > nesse o momento este arq1.txt está n estágio de Stage . Ou seja está pronto para ser "comitado"
8-  git status > git informa que o arq2.txt está untracked e que o arq1.txt está staged
9 - se vc quiser retornar o seu arquivo para o status de untracked (após usar git add .)> git restore --staged <file>..." to unstage)
10- se vc quiser ver o que foi alterado no arquivo>git diff --nomedoarquivo
11 - se quiser ver as alterações em termo de tempo de modificação>git log
12- se quiser ver qual o diretório remoto do seu repositório>git remote -v >>se não aparecer nada é porque não existe um repositório remoto
13 - se quiser fazer o link do repositório remoto para o repositório local >git remote add origin "nome do repositório remoto"
ex.  git remote add origin "https://github.com/cesarebteixeira/GitCompleto.git"

$ git remote -v
origin  https://github.com/cesarebteixeira/GitCompleto.git (fetch)
origin  https://github.com/cesarebteixeira/GitCompleto.git (push)
